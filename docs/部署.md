# 腾讯云轻量服务器（OpenCloudOS）部署流程

> 目标：把 mp-auto-publisher 部署到腾讯云轻量应用服务器，服务器 7×24 在线，本地电脑关机也能按时自动推送公众号文章。
>
> 假设：你已经买了腾讯云轻量服务器、装的是 OpenCloudOS、用 **OrcaTerm** 客户端能 root 登进去（OrcaTerm 是腾讯云官方 SSH 工具，左边连接管理器选实例，双击就能进终端，下方还有快捷命令、文件管理器）。

---

## 一、本地准备（在你自己的 Mac 上做）

1. **进项目目录**

   ```bash
   cd /Users/wangjunbin/mp-auto-publisher
   ```

2. **确认 .gitignore 已经生效**（仓库根目录已经有 `.gitignore`，敏感文件 `.env`、`data/`、`dist/`、`node_modules/` 全部被忽略）

   ```bash
   cat .gitignore           # 应该能看到 .env data/ dist/ node_modules/
   git ls-files | grep -E '^\.env|^data/' # 应该没输出，说明已被解除追踪
   ```

   > ⚠️ 如果是从老仓库迁过来，可能 `.env` 历史里有旧 secret。新建 GitHub 仓库前，**先把公众号 AppSecret / AI Key 重置一遍再填进 `.env`**，否则旧泄漏的还能用。

3. **把代码推到 GitHub**（推荐，后续更新代码只要 `git pull`）

   ```bash
   git add -A
   git commit -m "ready for tencent cloud deploy"
   ```

   如果还没建远程仓库，到 https://github.com/new 建一个 **私有仓库** `mp-auto-publisher`，然后：

   ```bash
   git remote add origin git@github.com:你的GitHub用户名/mp-auto-publisher.git
   git push -u origin main
   ```

   > **不想用 GitHub？** 跳过本步，用 scp 上传：
   > ```bash
   > tar czf /tmp/mp.tgz --exclude=node_modules --exclude=dist --exclude=data \
   >   -C /Users/wangjunbin mp-auto-publisher
   > scp /tmp/mp.tgz root@你的服务器公网IP:/root/
   > ```

## 二、腾讯云控制台开端口（最容易漏，先做）

腾讯云轻量服务器有**两层**防火墙：云后台的「防火墙规则」 + 服务器内的 firewalld。**云后台这层必须手动开**，否则浏览器永远 `ERR_CONNECTION_TIMED_OUT`。

服务器内的 firewalld 由部署脚本自动放行，这一节**只解决云后台那层**。

### 走法（腾讯云轻量）

4. 浏览器打开 <https://console.cloud.tencent.com/lighthouse/instance/index>
5. 在「**实例列表**」里点你的实例名进去（不是「重置」/「续费」那些按钮，**直接点机器名**）
6. 顶上是几个 tab：「概要 / 云硬盘 / 备份点 / 防火墙 / 快照 / SSH 密钥 / 监控」 → 点 **「防火墙」**
7. 表格右上角点 **「添加规则」**（如果看到「应用模板」也行，但不如自定义直观）
8. 弹窗里填：
   | 字段 | 值 |
   |---|---|
   | 应用类型 | **自定义** |
   | 协议 | **TCP** |
   | 端口 | **3030** |
   | 来源 | **0.0.0.0/0**（IPv4 全网，先用这个跑通；想锁就填你自己出口 IP） |
   | 策略 | **允许** |
   | 备注 | `mp-auto-publisher` |
9. 点「确定」。回到列表能看到一条新规则、状态正常。

### 走法（如果你买的是 CVM 不是轻量）

入口不一样：<https://console.cloud.tencent.com/cvm/securitygroup> → 找到实例绑定的**安全组** → 「入站规则」→「添加规则」，字段同上。

> 后面想配域名 + HTTPS 时，**再回到这一页**加两条规则：TCP `80`、TCP `443`。

## 三、服务器上拉代码（OrcaTerm 里操作）

打开 OrcaTerm → 左边「连接管理器」双击你的实例 → 右侧弹出终端、自动以 root 登入，**接下来都在这个黑框里敲命令**。OrcaTerm 中间那栏「文件管理器」可以用来事后看 `/root/mp-auto-publisher/data/` 里的 config 和日志，不用敲 `cat`。

10. **装 git**

    ```bash
    dnf install -y git
    ```

11. **配 GitHub SSH 密钥**（私有仓库推荐这种方式，比 PAT 省心；公开仓库可以跳过本步直接用 HTTPS）

    服务器上生成密钥：

    ```bash
    ssh-keygen -t ed25519 -C "你的GitHub邮箱"
    # 一路回车（passphrase 留空），输出会显示公钥在 /root/.ssh/id_ed25519.pub
    cat /root/.ssh/id_ed25519.pub
    ```

    复制输出整段（`ssh-ed25519 AAAA... 邮箱`），到浏览器：

    - <https://github.com/settings/keys> → 「New SSH key」→ Title 随便填如 `tencent-mp-server`，Key 粘贴，「Add SSH key」

    回服务器验一下：

    ```bash
    ssh -T git@github.com
    # 第一次会提示 yes/no，输 yes
    # 看到 "Hi 你的用户名! You've successfully authenticated" 就成了
    ```

12. **拉代码**

    - **SSH 方式**（推荐，配过密钥后 push/pull 不用再输密码）：

      ```bash
      cd /root
      git clone git@github.com:你的GitHub用户名/mp-auto-publisher.git
      cd mp-auto-publisher
      ```

    - **HTTPS + PAT 方式**（懒得搞 SSH 时备选）：

      ```bash
      cd /root
      git clone https://github.com/你的GitHub用户名/mp-auto-publisher.git
      cd mp-auto-publisher
      ```

      私有仓库会要账号 + Personal Access Token（不是登录密码）。PAT 生成入口：<https://github.com/settings/tokens> 勾 `repo` 权限。

    - **没用 GitHub，scp 上传 tar 包**：

      ```bash
      cd /root
      tar xzf mp.tgz
      cd mp-auto-publisher
      ```

## 四、一键部署脚本

脚本会自动：装 Node 20 + pm2 → 装项目依赖 → 开 firewalld 3030 → 用 pm2 启动 + 设置开机自启。

13. **设个登录密钥**（给网页 UI 登录用，别人不知道密钥就进不去）

    ```bash
    ACCESS_KEY='你的强密码123' bash deploy/server-setup.sh
    ```

    > `你的强密码123` 换成自己的，记下来后面要用。

14. 等脚本跑完，最后会打印这种信息：

    ```
    ============================================
      ✓ 部署成功
    --------------------------------------------
      公网 IP   : 1.2.3.4
      端口       : 3030
      访问地址   : http://1.2.3.4:3030
      登录密钥   : 你的强密码123
    ```

    **记下公网 IP**，下一步要用。

## 五、把公网 IP 加进微信公众号白名单

15. 浏览器打开 https://mp.weixin.qq.com，用**公众号管理员**登录
16. 左下角 **设置与开发** → **开发** → **基本配置**
17. 下方「公众号开发信息」找到 **IP 白名单** → 点 **查看**
18. **管理员微信扫码确认**
19. 填上一步的服务器公网 IP（**只填 IPv4，一行一个，不要带空格**），保存

> 公网 IP 是固定的，**这步只做一次**。

## 六、网页 UI 配置

20. 浏览器打开 `http://你的公网IP:3030`
21. 输入 **登录密钥**（第 13 步设的 `ACCESS_KEY`）
22. 进「**配置**」页，依次填：
    - 公众号 **AppID** + **AppSecret**（mp.weixin.qq.com → 设置与开发 → 开发 → 基本配置）
    - AI Provider + Key（DeepSeek / OpenAI / 其他）
    - Pexels API Key（用来配图，https://www.pexels.com/api/ 免费申请）
    - 保存
23. 进「**生成**」页，输个关键词（如"AI 编程"），点生成，确认能跑通、能推送草稿到公众号后台。

---

## 七、设定时任务（**核心**）

24. 进「**定时**」页 → 添加任务
25. 填关键词 + cron 表达式，示例：

    | 需求 | cron |
    |--|--|
    | 每天上午 9 点一篇 | `0 9 * * *` |
    | 每天 9 点和 18 点各一篇 | `0 9,18 * * *` |
    | 工作日 10 点 | `0 10 * * 1-5` |
    | 每 6 小时一篇 | `0 */6 * * *` |

26. **enable** 这个任务

> 服务器进程里内置 `node-cron`（`server.js:296`）会自动按点触发。**你本地电脑可以随便关机/断网，不影响服务器运行。**

## 十、常见问题

### Q：浏览器访问 `http://IP:3030` 一直转圈

按顺序排查：

```bash
pm2 list                        # 是否 online
ss -tlnp | grep 3030            # 端口是否在监听
firewall-cmd --list-ports       # firewalld 是否放行
```

如果上面都正常，**99% 是腾讯云控制台的防火墙规则没加** → 回第二步重新加 3030。

### Q：推送草稿报 `40164 invalid ip`

公众号 IP 白名单没填对。回第五步核对，**不要带空格、不要写错位**。

### Q：服务器重启后服务没自动起来

```bash
pm2 startup           # 看输出的 sudo 命令
sudo <复制上一行最后那条 systemctl 命令>
pm2 save
```

### Q：本地 Electron 版会和服务器冲突吗？

不会。`electron-main.js:11` 把 Electron 的数据目录指到了 `~/Library/Application Support/公众号自动发文/data/`，和服务器的 `/root/mp-auto-publisher/data/` 完全隔离。两边各自独立运行、独立配置。

### Q：怎么备份服务器数据

```bash
# 在本地 Mac 执行
scp -r root@你的服务器IP:/root/mp-auto-publisher/data ~/mp-backup-$(date +%F)
```

`data/` 里有所有配置、任务、历史。定期备份一下保险。

## 十一、（可选）配域名 + HTTPS

不想每次输 `IP:3030`？可以用域名 + Caddy 自动 HTTPS。

1. 阿里云/腾讯云万网买个域名（¥9/年起）
2. 加 A 记录：`mp.你的域名.com` → 服务器公网 IP
3. 腾讯云防火墙再开 TCP `80`、`443`
4. 服务器上装 Caddy（OpenCloudOS）：

   ```bash
   dnf install -y 'dnf-command(copr)'
   dnf copr enable -y @caddy/caddy
   dnf install -y caddy
   ```

5. 改配置：

   ```bash
   cp /root/mp-auto-publisher/deploy/Caddyfile.example /etc/caddy/Caddyfile
   sed -i 's/yourdomain.com/mp.你的域名.com/' /etc/caddy/Caddyfile
   systemctl enable --now caddy
   ```

完成后访问 `https://mp.你的域名.com` 即可，Caddy 自动签发 + 续期 Let's Encrypt 证书。

> 你已经装了 nginx，如果想用 nginx 代替 Caddy，可以告诉我，我给一份 nginx 反代 + certbot 的配置。
